Stdlib coverage matrix¶

Auto-generated. Do not hand-edit. Re-run cargo xtask stdlib-coverage after changing the support state of a module.

Columns:

  • Interp — gos run (bytecode VM + tree-walker fallback).
  • Compiled — gos build (Cranelift) and gos build --release (LLVM).
  • Tests — at least one integration test exercising the item.

Glyphs: ✓ supported · ◑ partial · ✗ missing.

How to regenerate this page¶

cargo xtask stdlib-coverage

How to interpret the columns¶

  • A module is marked Interp ✓ when at least one of its items resolves through gossamer-interp::builtins::install. A ◑ means some items resolve and some do not. A ✗ means none of the items are wired.

  • A module is marked Compiled ✓ when at least one of its items has a runtime symbol declared in gossamer-codegen-llvm/src/emit.rs::RUNTIME_DECLARATIONS and a Cranelift import in gossamer-codegen-cranelift/src/native.rs, or is dispatched as a method via gossamer-mir/src/lower.rs::lower_method_call.

  • A module is marked Tests ✓ when at least one integration / parity / phase test exercises it.
